# 🔧 服务商操作手册(贝壳自用)

这份文档只给贝壳看,别发给客户。

## 新客户接入流程 · 5 分钟做完

### Step 1. 把入口链接发给客户

从 [客户微信话术.md](客户微信话术.md) 选个模板(推荐模板 1 或 2),粘到微信发客户。

### Step 2. 等客户把公钥发来

客户会跑 `onboarding.sh`,到 Step 3 时屏幕会打印他的公钥。让客户把**一整段**(从 `ssh-ed25519` 开始到最后的 comment)发你微信。

公钥长这样:

```
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIExxxxx... ai-agent-客户电脑-20260423
```

### Step 3. 把公钥配到 4 个 repo

**最快方式(推荐)**:在你的 VSCode 终端(ai-studio 目录)跑:

```bash
bash scripts/add-client-deploy-key.sh "客户标识" "<粘贴客户公钥>"
```

(这个脚本下面第 4 节有,帮你一键配 4 个 repo)。

**手动方式**:

```bash
PUBKEY='ssh-ed25519 AAAA... ai-agent-客户电脑-20260423'   # 用单引号包住
CLIENT_TAG="客户简称-$(date +%Y%m%d)"

for repo in web-ops-public site-builder-public astro-b2b-starter-public mcp-servers-public; do
  echo "$PUBKEY" | gh repo deploy-key add - -R beikeint/$repo -t "$CLIENT_TAG"
done
```

### Step 4. 告诉客户"配好了"

微信发[客户微信话术.md](客户微信话术.md) 里的**模板 4**给客户,让他继续。

### Step 5. 等客户装完,远程验证

客户装完会告诉你"好了"。你可以远程进去(或让客户截图)看:

- `~/ai-agents/` 下应该有 4 个子目录
- 每个目录下有 `.git`(说明 clone 成功)
- 终端跑 `gh auth status`(如果客户装了 gh)看 Deploy Key 是否生效

## 客户侧常见问题 · 快速处理

| 症状 | 原因 | 处理 |
|---|---|---|
| `Permission denied (publickey)` | Deploy Key 没配好 / 公钥粘错 | 重新要客户发公钥,检查有没有换行符或空格被吞 |
| `Could not resolve host: github.com` | 客户防火墙 / DNS | 让客户换手机热点试一下 |
| `onboarding.sh: command not found` | 客户 curl 没装或权限问题 | 让客户复制脚本内容到文件里手动跑 |
| MCP 在 Claude Code 里不显示 | `~/.claude/mcp.json` 格式错 | 远程进去帮他写正确的 mcp.json |
| 某个智能体跑不动 | `.env` 没填 API key | 远程进去帮他填,或引导他自己申请后填 |

## 客户离职/流失 · 撤销访问

```bash
# 查这个客户的所有 Deploy Key id(按标题前缀匹配)
CLIENT_TAG="客户简称"

for repo in web-ops-public site-builder-public astro-b2b-starter-public mcp-servers-public; do
  echo "=== $repo ==="
  gh repo deploy-key list -R beikeint/$repo --json id,title \
    | jq -r ".[] | select(.title | startswith(\"$CLIENT_TAG\")) | \"\(.id) \(.title)\""
done

# 确认无误后,删除
for repo in web-ops-public site-builder-public astro-b2b-starter-public mcp-servers-public; do
  gh repo deploy-key list -R beikeint/$repo --json id,title \
    | jq -r ".[] | select(.title | startswith(\"$CLIENT_TAG\")) | .id" \
    | xargs -I {} gh repo deploy-key delete {} -R beikeint/$repo
done
```

撤销后,客户本地的 repo 还在,但 `git pull` 会失败(没有读权限)。

## 一键加 Deploy Key 脚本(放在 ai-studio/scripts/)

```bash
#!/usr/bin/env bash
# scripts/add-client-deploy-key.sh
# 用法:bash scripts/add-client-deploy-key.sh "客户标识" "<公钥>"

set -euo pipefail

CLIENT="${1:-}"
PUBKEY="${2:-}"

if [ -z "$CLIENT" ] || [ -z "$PUBKEY" ]; then
  echo "用法: $0 <客户标识> <完整公钥>"
  echo "示例: $0 张三公司 'ssh-ed25519 AAAA... comment'"
  exit 1
fi

TAG="${CLIENT}-$(date +%Y%m%d)"

for repo in web-ops-public site-builder-public astro-b2b-starter-public mcp-servers-public; do
  echo "配置 $repo..."
  echo "$PUBKEY" | gh repo deploy-key add - -R beikeint/$repo -t "$TAG"
done

echo "✅ 完成:4 个 repo 都配好了 $TAG"
echo "现在可以告诉客户继续安装"
```

## 完整交付流程时间线参考

| 阶段 | 谁 | 做什么 | 大约时长 |
|---|---|---|---|
| ① | 贝壳 | 发话术 + 链接给客户 | 1 分钟 |
| ② | 客户 | 装 VSCode / Node / Claude 订阅 | 10-20 分钟 |

| ⑤ | 客户 | 继续 onboarding 下载 + 装依赖 | 5-10 分钟 |
| ⑥ | 贝壳 | 远程/微信指导客户配 mcp.json 和 .env | 10-20 分钟 |
| ⑦ | 贝壳 | 教客户第一个对话(如"每日巡检") | 5 分钟 |
| **合计** | - | - | **35-60 分钟** |
